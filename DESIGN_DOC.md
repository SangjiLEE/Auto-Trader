## 1. Problem Statement

개인이 사용할 자동매매 웹 시스템을 만든다. 한국주식 + 미국주식을 기술적 지표(이평선, 일목균형, RSI, 거래량) 기반으로 매매하되, **양의 기댓값(positive expected value)이 확인된 전략만** 실거래에 투입한다.

**사용자 프로필:**
- 주식 투자 경력 5년 이상
- React/TypeScript 개발자, Python 초보 (의향 있음)
- 자동매매 운영 경험 없음

**목표 구조:**
- **지향(Long-term)**: 점점 수익률이 개선되는 개인용 수익 파이프라인
- **중간(Mid-term)**: 월 30만원 수준 수익 (AI 구독료 충당) — 단, 원금 100만원에서는 12~18개월 후 자본 증액 시 현실화
- **단기(Short-term)**: **손해 가능성을 체계적으로 줄이고, 첫날부터 양의 기댓값을 가진 시스템 운영**

**출발 자본**: 100만원

---

## 2. What Makes This Cool

- 리테일 개인이 퀀트 인프라를 직접 만드는 건 소프트웨어 공학 + 금융 공학 + 심리학의 교차점. 재미있는 영역.
- "검증 → 개선 → 재검증" 반복에서 **돈과 기술 둘 다 복리로 쌓임**.
- 5년 투자 직관을 "기계적 시스템"으로 번역하는 작업 자체가 학습.
- 성공하면 평생 자산 (코드 + 시스템 + 자본). 실패해도 스킬은 남는다.

## 3. Constraints

| 항목 | 제약 |
|-----|------|
| 자본 | 100만원 |
| 시간 | 본업 병행 (월 10~20시간 예산) |
| 플랫폼 | 웹앱 (React 선호) + Python 백엔드 |
| 브로커 | 한국투자증권 KIS Open API (확정) |
| 법적 | 개인 사용 전용, 타인 자금 미취급 |
| 실행 환경 | 24/7 가동 가능한 서버 (클라우드 또는 로컬 머신) |

**비용 구조 (설계에 반영 필수):**

| 항목 | 비용 |
|-----|------|
| 국내주식 수수료 (KIS 비대면) | 매매당 약 0.015% |
| 증권거래세 (매도) | 2026년 기준 약 0.18% |
| 미국주식 수수료 | 매매당 약 0.25% |
| 환전 수수료 | 일반 약 0.5% (환전우대 적용 시 0.1% 이하) |
| 해외주식 양도세 | 연 250만원 초과 수익의 22% |
| 배당 원천징수 | 15.4% |

고빈도 매매는 이 비용 구조 때문에 수학적으로 불리함.

## 4. Premises

1. **양의 EV 우선**: 불확실한 고수익보다 검증된 저수익.
2. **소규모 시작**: 100만원은 "학비". 검증 후 증액.
3. **저빈도 전략**: 비용 구조상 고빈도 불리.
4. **리스크 제어 최우선**: 파산 리스크 0에 근사.
5. **단순함 유지**: 이해 못 하는 전략은 운영 못 한다.
6. **개선은 수치로 측정**: "느낌 좋다"로 의사결정하지 않는다. Sharpe, MDD, 승률 등 지표로 판단.
7. **엣지는 지표가 아니라 규율에서 온다**: 리테일이 이길 수 있는 영역은 리스크 관리와 실행 규율.

## 5. Approaches Considered

### A. 장기 트렌드 추종 (Trend Following)

**개요**: 가격이 장기 이평선 위면 보유, 아래면 현금. 자산 클래스: 주식 지수 ETF.

**참고 연구**: Meb Faber "A Quantitative Approach to Tactical Asset Allocation" (2007, 업데이트 2013). 1973~2012 데이터에서 주식 프리미엄 유지하면서 MDD를 절반으로 감소.

**100만원 구현**:
- 자산: KODEX 200 (KR), SPY/QQQ (US)
- 시그널: 월말 종가 > 10개월 이평선 → 보유, 아니면 현금
- 매매: 월 1~2회 → 비용 무시 가능
- 기대 수익: 연 8~12%, MDD 15~20%

**사용자 지표 활용도**: 이평선(200일 또는 10개월) 그대로 사용.

**장점**: 검증 확실, 구현 쉬움, 심리적 부담 적음.
**단점**: 횡보장 whipsaw, 뉴스/이벤트에 뒤처짐.
**EV**: 양수 (long-term, 광범위 검증).
**추천도**: ★★★★★

### B. 듀얼 모멘텀 (Dual Momentum)

**개요**: 여러 자산군 중 상대적으로 가장 강한 것 보유 (relative momentum) + 절대 모멘텀이 현금보다 강할 때만 진입 (absolute momentum).

**참고 연구**: Gary Antonacci "Dual Momentum Investing" (2014). 1974~2013 데이터에서 S&P 500 대비 연 4% 초과 수익, MDD는 절반.

**100만원 구현**:
- 자산 유니버스: 한국 지수 ETF, 미국 지수 ETF, 미국 채권 ETF (TLT), 현금
- 월 1회 리밸런싱: 지난 12개월 수익률 기준
- 기대 수익: 연 10~15%, MDD 20% 미만

**사용자 지표 활용도**: 12개월 가격 변화율. 이평선 보조.

**장점**: 학술 검증, 단순, 자산 배분 효과 내장.
**단점**: 2022년처럼 주식·채권 동반 하락 시 방어 약함.
**EV**: 양수.
**추천도**: ★★★★★

### C. 스윙 트레이딩 (Swing Trading)

**개요**: 며칠~몇 주 보유. 이평선 풀백, 일목 전환선/기준선 크로스, 구름대 돌파, 박스권 돌파.

**100만원 구현**:
- 자산: 유동성 큰 개별주 (삼성전자, 테슬라 등) 또는 지수 ETF
- 매매: 주 1~5회 → 비용 영향 의미 있음
- 기대 수익: 연 10~30% (전략/시장에 따라 변동성 큼)

**사용자 지표 활용도**: **가장 높음.** 이평선(20/60/120/200) + 일목균형(9/26/52 + 선행/후행 스팬) + RSI + 거래량을 조합 신호로 사용.

**장점**: 재미있고 지표 실험 공간 넓음. 사용자 직관 활용 가능.
**단점**: 시장 체제 변화 민감, 백테스트 과적합 위험 큼.
**EV**: 조건부 — 세팅에 따라 크게 달라짐.
**추천도**: ★★★

### D. 변동성 돌파 (Volatility Breakout)

**개요**: 전일 고가~저가 범위의 일정 비율을 당일 시가에 더한 값 돌파 시 매수. 당일 청산.

**참고**: Larry Williams "The Secret of Selecting Stocks" (1965). 한국 시스템 트레이더 커뮤니티에서 많이 다룸.

**100만원 구현**:
- 자산: 지수 ETF (KODEX 레버리지 등) 또는 유동성 큰 대형주
- 매매: 일 1회
- 기대 수익: 연 5~15% (과거엔 높았으나 퀀트들이 많이 먹어서 약해짐)

**사용자 지표 활용도**: 거래량을 필터로 사용. 이평선은 추세 필터.

**장점**: 구현 간단, 한국 커뮤니티 자료 많음.
**단점**: 전략 크라우딩으로 엣지 감소, 슬리피지 크면 EV 음수 전환.
**EV**: 약한 양수 ~ 제로 근처.
**추천도**: ★★★

### E. 평균 회귀 (Mean Reversion)

**개요**: 과매도에서 매수, 과매수에서 매도. RSI + Bollinger Band + 거래량 스파이크 조합.

**100만원 구현**:
- 자산: 지수 ETF 또는 우량주
- 시그널 예: RSI < 30 + 종가 < 볼린저 하단 + 거래량 평소의 2배 → 매수, RSI > 50 또는 N일 경과 시 청산
- 매매: 주 1~3회

**사용자 지표 활용도**: **매우 높음.** RSI + 거래량이 핵심 지표.

**장점**: 한국 시장 박스권 특성과 궁합 좋음.
**단점**: 강한 트렌드 마켓(2020, 2024)에선 큰 손실 (떨어지는 칼날 잡기).
**EV**: 조건부 (regime detection 필요).
**추천도**: ★★★

### F. 페어 트레이딩 (Pair / Statistical Arbitrage)

**개요**: 상관 높은 두 종목의 스프레드가 벌어지면 상대적으로 비싼 쪽 매도 + 싼 쪽 매수. 시장 방향과 무관(시장 중립).

**후보 페어**: 삼성전자 vs SK하이닉스, 현대차 vs 기아, KODEX 200 vs KOSEF 200 등.

**100만원 구현**:
- 2 포지션 동시 운용 → 포지션당 약 50만원
- 주 1~2회 진입/청산
- 기대 수익: 연 5~10%, 낮지만 MDD 작음

**사용자 지표 활용도**: 낮음. 대신 통계 기법(cointegration, z-score) 학습 필요.

**장점**: 시장 방향 리스크 제거, 드로우다운 작음.
**단점**: 상관 깨지면 양쪽 다 손실, 통계 학습 필요, 공매도는 개별주 불가 → 역페어 ETF 활용 등 우회 필요.
**EV**: 양수 (페어 선정과 정확한 cointegration 검증 시).
**추천도**: ★★ (Tier 3, 나중에 도전)

### G. 단타 / 데이 트레이딩

**100만원 현실 계산**:
- 회당 비용: 수수료 0.015%×2 + 거래세 0.2% + 슬리피지 0.05~0.1% ≈ **0.3%**
- 일 3회 매매 = 일 0.9% 비용 → 연 225거래일 × 0.9% = **연 202% 비용**
- 이걸 이기려면 매매 성공률과 수익폭이 비현실적으로 높아야 함.

**연구 결과**:
- FINRA (미국): 데이트레이더 80% 첫 해 손실.
- Barber et al. (Taiwan, 2014): 장기 데이트레이더 80% 순손실, 나머지 20%도 시급 환산 최저임금 미달.

**EV**: **음수 확정 (100만원 수준).**
**추천도**: ★ (추천하지 않음)

### H. 스캘핑

초~분 단위. 리테일 100만원에서는 속도/수수료 모두에서 HFT에 압도됨. **완전 제외.**

## 6. Recommended Approach

**Tier 시스템 단계적 접근:**

### Tier 1 — 엔진 + 기본 전략 (0~6개월)

- **전략 A (장기 트렌드 추종)** + **전략 B (듀얼 모멘텀)** 동시 구현
- 가장 단순하면서 양의 EV가 학술적으로 확립된 2개
- 이 시기 핵심 목표: **시스템 인프라 구축 + 백테스트 정확도 확보**
- 배포: 모의투자 3개월 이상 → 실거래 소액 (예: 10~20만원)

**왜 이 두 개?**
1. 둘 다 저빈도 → 비용 영향 무시 가능 → 100만원에서도 현실적
2. 과적합 리스크 낮음 (파라미터 거의 없음)
3. 구현 간단 → 엔진·데이터·리스크 관리 코드에 집중 가능
4. 실패해도 "전략 문제"가 아니라 "구현 문제"로 원인 분리 쉬움

### Tier 2 — 엣지 확장 (6~12개월)

Tier 1 시스템이 안정되면:
- **전략 C (스윙)** 또는 **전략 E (평균 회귀)** 중 하나 추가 구현
- 사용자님이 요청한 지표 조합(이평선/일목/RSI/거래량)을 여기서 본격 활용
- 각 전략을 별도 "슬롯"으로 운영, 자본을 나눠 할당 (예: Tier1 70%, Tier2 30%)
- 측정: 각 전략별 Sharpe, 최대낙폭, 상관관계 추적

**엣지 개선 루프:**
```
전략 가설 → 백테스트 → 모의투자 → 소액 실거래
     ↑                                      ↓
     └─── 엣지 decay 감지 시 재설계 ────────┘
```

### Tier 3 — 고급 전략 + 자본 증액 (12개월+)

- **전략 F (페어 트레이딩)** 도전 (시장 중립 학습)
- Tier 1~2 실거래 성과가 양수 확정되면 자본 증액
- 멀티 전략 포트폴리오 관리 (Risk Parity 식 배분)

**단타(G), 스캘핑(H)**: 전 기간 제외. 100만원~1억 수준에서 음수 EV.

## 8. Open Questions

1. **KIS 모의투자의 현실성**: 모의투자 체결이 실거래와 얼마나 다른가? 슬리피지/부분체결 모델링 필요성 검토.
2. **세금 처리**: 해외주식 양도세 250만원 공제, 배당세, 양도세 계산을 백테스트와 손익 추적에 어떻게 반영할 것인가.
3. **한국/미국 시장 시간 차이**: 한국 장(09:00~15:30 KST) 마감 후 미국 장(22:30~05:00 KST) 스케줄 관리. 수면 중 돌아갈 로직 안전성.
4. **전략 성능 저하 감지**: 엣지 decay monitoring — 한 때 먹히던 전략이 안 먹힐 때 어떻게 감지하고 자동/수동으로 끌 것인가.
5. **환전 타이밍**: 미국주식 매매 시 환전 비용 관리. 일괄 환전 후 운용 vs 매매 시마다 환전.
6. **데이터 소스 이중화**: KIS API 시세 외에 FinanceDataReader, yfinance 등 보조 소스 필요성.

## 13. 솔직한 리스크

1. **엣지 착각 (Overfitting)**: 백테스트 잘 나온 전략이 실거래에서 무너지는 게 가장 흔한 실패. walk-forward와 out-of-sample 검증으로 완화하되 완전 제거 불가.
2. **심리 이탈**: 수익 나면 자본 증액이 급해지고, 손실 나면 끄고 싶어진다. 자동화의 의미가 깨짐. **사전에 정한 규칙에서 벗어나지 않는 것이 최대 도전**.
3. **API 장애**: 증권사 API 다운·지연 시 포지션 보호 불가. Failover 설계(정지 상태 유지, 포지션 플랫 유지)가 필요.
4. **세금 폭탄**: 해외주식 양도세 250만원 공제 초과 시 22% 세율. 수익이 클수록 실수익이 많이 깎임.
5. **시간 소모**: 코드 쓰는 재미에 빠지면 투자 대신 개발에만 시간. "AI 구독료 벌려다 본업 연봉 손해" 역설 주의. 월 10~20시간 이상 쓰면 경고등.
6. **100만원 한계**: Tier 1 전략 운영에는 충분하나, 포지션 분산에 제약. 지수 ETF 중심 운영이 강제됨 (오히려 단순함이 장점이 되는 역설).

## 14. What I noticed about how you think

세션에서 관찰한 사용자 특성 (개발자/투자자/창업자 관점):

1. **비현실적 목표(주 10%)를 빠르게 조정함**. 수치 피드백 받고 저항 없이 재설정. 이건 평균 이상의 메타인지. 대부분의 리테일 투자자는 "하지만 내 전략은 다르다"로 밀어붙임.

2. **"손해 보면 6개월 후 수익난다"는 제 framing에 즉각 반박**. "손해를 보았을 때 6개월 이후에 이득을 본다는 확률이 높아져? 아니잖아"라는 답은 **정확히 퀀트 사고**. 시간이 아니라 엣지가 개선 요인임을 직관으로 파악.

3. **"손해 가능성을 계속 줄여서 수익 나게 만들고 싶다"** 표현은 이 세션 전체의 정답. 이게 바로 기댓값 최대화의 본질. 많은 초보가 이해 못 하는 프레임.

4. **솔루션(지표 조합)을 먼저 떠올리고 문제 정의는 나중에 내는 패턴**. 개발자 흔한 경향. 이 Design Doc이 "문제 → 솔루션" 순서로 재정렬하는 작업. 다음 프로젝트부터는 이 순서가 몸에 배게 하면 좋음.

5. **Python 배울 의향 + 코딩 에이전트 활용**. 도구를 바꿀 수 있는 유연성. 이게 없으면 이 프로젝트 진행 불가.

### 추천 태도

- **100만원 = 학비**: 손실이 나도 "공부값"으로 치환. 1년 후 남은 것은 잔고가 아니라 **운영 가능한 시스템 + 검증된 전략 + 본인의 퀀트 직관**.
- **엣지 측정 지표로 의사결정**: "수익률" 대신 "Sharpe, profit factor, 엣지 안정성" 같은 지표 기준으로 판단. 단기 운 vs 실제 엣지 구분이 이걸로 가능.
- **월 시간 예산**: 10~20시간. 넘으면 빨간불. 본업·생활 우선.
- **3년 관점**: 이 프로젝트는 1개월 프로젝트가 아님. 3년 동안 시스템 + 자본 + 기술을 복리로 쌓아가는 관점으로 접근.
