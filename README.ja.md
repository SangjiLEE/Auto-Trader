# Auto Trader

🌐 **言語**: [한국어](README.md) · [English](README.en.md) · **日本語**

> **8日間で一人で作ったマルチ戦略自動売買システム。**
> Python + Claude Code（AIペアプログラミング）+ KIS Open API。

平日に韓国株と米国株を自動売買。市場レジーム（BULL / RANGE / BEAR）を毎日判定して、レジームごとに違うルールを適用。Fear & Greed の極端値で分割発注。すべての約定は Telegram にリアルタイム通知。

**現在は紙トレード（模擬投資）口座で稼働中**(本番運用に切り替える前に 3〜6 ヶ月の検証期間)。

## なぜ作ったか

### 当初の目標
> 「AI サブスク料金(月 30 万ウォン)を自動売買の利益で賄いたい。」

### 本当の目標(検証後に修正)
> 「**損失の可能性を体系的に減らしながら、徐々に収益性を改善していく個人用の収益パイプライン。**」

8 日間データから学んだこと:
1. **100 万ウォンで月 30 万ウォン稼ぐのは数学的に不可能**(必要月利 30%)
2. **すべての戦略が単純な Buy & Hold に絶対値で勝てない**(長期強トレンド市場で)
3. **しかし MDD(最大ドローダウン)は半分以下に減らせる**
4. → "うまく勝つ" より "うまく負けない" 方が現実的

### 結論
- 資本が小さいうちは **学習 + インフラ整備のコスト**として受け入れる
- **6〜12 ヶ月の紙トレード検証**後に増資を再検討
- システムをきちんと作っておけば、**3 年後に資本が増えた時点ですぐに適用可能**

## どう作ったか

### 8 日間の進化(意図的に段階を踏んだ)

| 日 | マイルストーン | テーマ |
|----|---------------|------|
| 1 (日) | KIS 口座開設 + API キー発行 + 環境構築 | インフラ |
| 2 (月) | OAuth2 認証、Hello World、残高/相場照会 | API 基礎 |
| 3 (火) | DM バックテスト + **頑健性検証 3 種**(パラメータ、OOS、年度別) | データ検証 |
| 4 (水) | **初の紙トレード約定**(069500 × 508 株) + Telegram 統合 | 初の発注 |
| 5 (木) | 日次自動レポート + 米国市場対応 | 自動化 |
| 6 (金) | 複数銘柄バックテスト + Enhanced Swing v3(レジーム適応) | 戦略の進化 |
| 7 (土) | F&G 統合 + v4(極端値の分割発注) | 心理シグナル |
| 8 (日) | GitHub プッシュ + ドキュメント整備 | 整理 |

各段階で **バックテスト → 頑健性検証 → ドライラン → 本発注** の 4 段階ゲートを通している。バックテストで効果が薄かった戦略(短期スキャル、RSI / BB 平均回帰)は廃棄 — データが教えてくれた。

進化の詳細: [docs/JOURNEY.md](docs/JOURNEY.md)

### 開発スタイル: Claude Code + AI ペアプログラミング

**Claude(Anthropic)とペアプロ。**

- **ユーザー**: 意図、ドメイン知識、意思決定、検証
- **Claude**: コード作成、デバッグ、バックテスト設計、テスト
- **検証ループ**: すべてのコードはユーザーが実行し、結果を確認してから次の決定へ

このループで 8 日間に約 7,000 行 / 30+ モジュールを構築。**AI がコードを書き、人間が意思決定と検証を持つ。**

### ハーネスエンジニアリング(Harness Engineering)

このシステムの肝は **再利用可能な検証インフラ(= ハーネス)**。

詳細: [docs/HARNESS.md](docs/HARNESS.md)

要約:
1. **バックテストハーネス × 5** — 同じ形なので新戦略の検証が速い
2. **頑健性検証ハーネス** — パラメータスイープ + IS/OOS 分割 + 年度別を自動で回す
3. **戦略プラグインアーキテクチャ** — strategy モジュールを差し替えるだけで別のルールを試せる
4. **レジーム分類ハーネス** — BULL / RANGE / BEAR の単一の真実
5. **ドライラン ↔ 実行 トグル** — `--execute` フラグ 1 つで安全モードと本番モードを分離
6. **状態再構成ハーネス** — DB の `trades` テーブルからポジション・レジーム・部分利確の状態をすべて復元
7. **5 つの launchd ジョブを分散** — 1 つが失敗しても他には影響しない

このハーネスのおかげで、8 日間で **5 回の大きな戦略再設計**(slow → fast → v3 → v4 → relaxed v4)が可能になった。

→ **VAA が全指標で DM を上回る。** Day 9 の最大の発見。

## クイックスタート

git clone https://github.com/SangjiLEE/Auto-Trader.git
cd Auto-Trader
python3.12 -m venv .venv
source .venv/bin/activate

cp .env.example .env  # KIS API キー、Telegram トークンを記入
